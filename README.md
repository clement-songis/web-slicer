# web-slicer

A **multi-user web 3D slicer** aiming for **full parity** with
[OrcaSlicer](https://github.com/SoftFever/OrcaSlicer): the same 858 parameters,
the same settings organization (21 tabs, 100 sections), the same 11,895 system
presets, complete 3D preparation, slicing, G-code preview, and sending to
Klipper (Moonraker) printers — in the browser, with isolated accounts.

> The slicing engine wraps OrcaSlicer's **libslic3r** (`vendor/OrcaSlicer`,
> read-only): same algorithms, same G-code.

## Architecture (3-layer monorepo)

| Layer | Directory | Stack | Role |
|---|---|---|---|
| Engine | [`engine/`](engine/README.md) | Rust + cxx-FFI bridge | `SlicerEngine` trait, a mirror of libslic3r (generated parameter registry, slicing, G-code parsing) |
| Backend | `backend/` | Rust · axum 0.8 · sqlx (SQLite/Postgres) · tower-sessions · argon2 | pure domain + adapters (storage, files, Moonraker); HTTP/WebSocket API; slicing queue |
| Frontend | `frontend/` | SvelteKit 2 · strict TS · Tailwind · Threlte/Three.js | parity UI; 3D scene; preview; API types generated by ts-rs |

Non-negotiable principles: [`.specify/memory/constitution.md`](.specify/memory/constitution.md)
(3-layer monorepo, `SlicerEngine` trait, `Storage` trait, backend/engine TDD,
fully traceable parity). Code identifiers are in **English**, comments in
**French**.

## Requirements — Nix environment

All tooling (Rust, Bun, clang/cmake/ninja, `orca-slicer`, `$LIBSLIC3R_DIR`,
Playwright browsers) is provided by the flake. **No manual installation.**

```sh
nix develop            # enter the devShell (tool banner printed on startup)
```

The v1 engine links against the Nix `libslic3r-headless` static libraries (FFI
bridge). Verify the C++ link chain:

```sh
nix build .#libslic3r      # → result/lib/*.a + include/
nix build .#dump-config    # validates the full link
./result/bin/dump-config   # dumps the runtime print_config_def (parity, SC-003)
```

FFI/CLI details: [`engine/README.md`](engine/README.md).

## Getting started

```sh
# Backend — http://localhost:3000 (SQLite in ./data by default)
cargo run -p backend

# Frontend — http://localhost:5173 (proxies /api → backend)
cd frontend && bun install && bun dev
```

On first startup the **11,895 system presets** (65 vendors) are seeded
automatically. The **first account created becomes the administrator**.

On first login a **setup wizard** lets you pick the printers you own (grid
ported from OrcaSlicer's "Select a printer" dialog); a **Moonraker connection is
optional** (a printer can be owned without being reachable over the network) and
can be added later from the printers management page. Printer selection in the
editor and slicing is restricted to the printers you own.

## Quality (constitution gates)

```sh
# Rust (engine + backend)
cargo fmt --check && cargo clippy --workspace -- -D warnings && cargo test --workspace

# Frontend
cd frontend && bun run lint && bun run check && bun run test

# Parity (regenerates inventories, resyncs the annexes, checks the matrix)
python3 audit/run_all.py
python3 audit/generate_parity_annexes.py
python3 audit/check_traceability.py
```

Atomic commits per task; TDD for the backend and the engine; every parity gap
goes through [`specs/001-orcaslicer-web-parity/exclusions.md`](specs/001-orcaslicer-web-parity/exclusions.md),
never by omission (the check fails otherwise).

## Features (parity by domain)

- **Accounts & isolation** — sign-up/instance policy, sessions, roles; any other
  user's resource is a **404** (never 403), with an automated isolation campaign
  (SC-008 at 100%).
- **3D preparation** — STL/3MF/STEP/OBJ imports **decoded server-side**
  (`engine-worker` process, binary WSMh display mesh; the frontend ships no
  parser — see reversal R7), Threlte scene, 16 gizmos,
  move/rotate/scale/cut/arrange/orient, multi-plate, mesh repair.
- **Settings** — 21 tabs / 100 sections / 525 faithful option rows
  (simple/advanced/expert modes), derivable presets with inheritance and overrides.
- **Slicing & queue** — multi-worker queue, automatic recovery, real-time
  progress over WebSocket; G-code / 3MF export.
- **G-code preview** — typed layers, 7 colorations, sliders, stats; lazy
  window-based loading under a memory budget (SC-007).
- **Printing (Moonraker)** — printer declaration (API key encrypted at rest),
  connection test, upload (start now), live status tracking,
  pause/resume/cancel.
